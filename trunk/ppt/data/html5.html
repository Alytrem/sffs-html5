HTML5

Présentation, démonstration et évaluation

Quentin Decré - quentin.decre@gmail.com

Introduction

  • HTML5 : le successeur de HTML4 et XHTML1
  • Fruit du travail du W3C (World Wide Web Consortium) et du WHATWG (Web Hypertext Application Technology Working Group).
  • Les objectifs :
    1. Unification du comportement entre les navigateurs
    2. Normalisation
    3. Développement de nouvelles fonctionnalités HTML, CSS et javascript
  • HTML5 est encore une ébauche, mais déjà supportée par beaucoup de navigateurs.

HTML4 : bilan

HTML4 permet déjà de faire beaucoup de choses. Les applications web n'ont pas attendu HTML5 pour naitre. Les webmails ou éditeurs de documents en ligne (office web apps, google docs...) deviennent très évolués, le chargement dynamique avec Ajax devient usuel, les API comme Jquery fleurissent et rendent le developpement plus rapide et robuste.

HTML5 n'est donc pas une récolution, mais une belle évolution.

HTML5 : plus que de l'HTML

HTML5 ~= HTML5 + CSS3 + JS

La timeline HTML5

  • Mai 2011 : Last Call (dernier appel à commentaires avant de figer les spécifications.
  • 2014 : fin de la définition du standard (tous les objectifs de compatibilité seront alors atteints
  • Mais rien n'empêche son utilisation dès maintenant. Il faut seulement prendre les précautions nécessaires.

Les objectifs de la présentation

  • Présenter les nouveautés HTML5, et faire un retour d'expérience
  • Evaluer les nouveautés HTML5 sur différents critères :
    • Maturité : indique le dégré d'évolution possible avant 2014. Plus c'est mature, moins c'est susceptible de changer.
    • Intérêt : mesure l'apport en terme de fonctionnalités (subjectif)
    • Sécurité : mesure des possibles problèmes de sécurité (données accessibles en local par un tiers, données en clair sur le réseau, données accessibles par un autre site, donne un accès au système).
    • Compatibilité (Mobile, IE, Firefox, Chrome, Safari) : la note est maximale si les 3 dernières versions majeures le supporte, puis perd 25% par versions ne le supportant pas, et tombe à 0 si la prochaine version ne le supportera pas non plus.
  • Présenter les démonstrations de chaque fonctionnalités
  • Faire un point sur l'aspect technique de la mise en oeuvre des fonctionnalités

Nouvelles balises

On peut classer les nouvelles balises en 3 groupes.

  • Les balises ayant pour but d'enrichir la sémantique.
  • Les balises ajoutant des fonctionnalités
  • Les nouveaux inputs (champs de formulaires)

Enrichissement de la sémantique

<div id="header">
<div class="article">
...

Voici le genre de choses que l'on peut voir fréquemment. Seulement, un div ne veut rien dire. Arrivent donc de nouvelles balises porteuses de sens.

L'autre nouveauté est que HTML5 ne fonctionne pas avec une hierarchie de titre (de h1 à h6) comme HTML4, mais avec une imbrication de sections, chaque section faisant repartir les titres à 1 en son sein.

Note : vous pouvez cependant toujours continuer à utiliser du code comme ci-dessus.

Les balises

On trouve ainsi les balises :

  • header/footer : une en-tête/pied d'une section ou d'un document.
  • hgroup : sorte de wrapper pour un ou plusieurs titres (h?) en lien (comme un titre et un sous-titre).
  • section : définition d'une nouvelle section.
  • article : similaire à section, mais plus riche sémantiquement.
  • nav : indique un menu
  • aside : indique une sidebar
  • figure
  • figcaption
  • time : indique une date de deux façons; pour l'ordinateur et pour l'homme
  • mark : met en valeur du texte

Exemple

<h2>Les articles</h2>
<nav>
    <ul>
        <li>Accueil</li>
        <li>Détails</li>
        <li>Contacts</li>
    </ul>
</nav>
<article>
    <header>
        <hgroup>
            <h1>Capgemini et HTML5</h1>
            <h2>Une percée prometteuse ?</h2>
        </hgroup>
    </header>
    <time datetime="2011-07-07">aujourd'hui</time>
    <p>Blah blah blah</p>
</article>

Nouveaux inputs

Nouveaux inputs

Arrivent également de nouveaux attributs / options

  • autocomplete : pour autoriser ou non l'auto-completion par le navigateur
  • autofocus : pour placer le curseur dans ce champ au chargement
  • list : pour lier à une datalist pour l'auto-completion
  • placeholder : pour afficher un texte qui se retire lors du focus
  • pattern : pour assurer que le contenu respecte un pattern
  • required : pour que le champ soit obligatoire

Tout ceci permet au navigateur (particulièrement mobiles) d'offrir un comportement plus adaptés au type de champ. Cela permet aussi une uniformisation du comportement en le déleguant au navigateur plutôt qu'au développeur.

Nouvelles relations

On retrouve de nouvelles valeur pour l'attribut rel (relation), des balises a et link. En voici une liste non-exhaustive.

  • archives : collection d'archives
  • author : indique plus d'informations sur l'auteur
  • external : lien externe (vers un autre site)
  • license : un lien vers les termes de la licence sous laquelle est publié le document

Pourquoi ? C'est tout simplement dans le but de standardiser ces valeurs d'attributs. En effet, il est possible pour chacun d'utiliser ses propres valeurs, mais il est toujours préférable de proposer les principales valeurs comme référence.

Microdata

Les microdatas permettent d'annoter le DOM (Document Object Model : les objets décrits par les balises), avec des paires nom/valeur issues d'un vocabulaire personnalisé.

Problème : on peut représenter du texte, tableaux..., mais pas des personnes, factures ou autre.

Solution : utiliser les micro-datas et du vocabulaire personnalisé pour permettre cette représentation

Pourquoi : pour améliorer l'interaction avec les navigateurs (pouvoir enregistrer un contact en vCard, un évènement...), ou avec les moteurs de recherche.

Mode d'emploi

  1. Trouver le vocabulaire qui nous intéresse sur www.data-vocabulary.org si il a déjà été défini (Person, Event...) ou définir son propre vocabulaire.
  2. Déclarer le type et le scope (portée) du vocabulaire. Exemple : 
    <section itemscope itemtype="http://data-vocabulary.org/Person">
  3. Pour chaque valeur, indiquer son nom : 
    <h1 itemprop="name">Quentin Decré</h1>

Un exemple complet

<section itemscope itemtype="http://www.data-vocabulary.org/Person/">
	<h1 itemprop="name">Quentin Decré</h1>
	<h2 itemprop="title">Etudiant ingénieur INSA</h2>
	<a itemprop="url" href="www.quentin-decre.fr">Mon site</a>
	<p itemprop="address"
		itemscope itemtype="http://data-vocabulary.org/Address">
		<span itemprop="locality">Cesson-Sévigné</span>,
		<span itemprop="postal-code">35510</span>
		<span itemprop="country-name">France</span>
	</p>
</section>

Note : La propriété "adress" d'une personne est en une propriété complexe décrite par une autre micro-data.

Micro-data : Evaluation

  • Intérêt : 20%
  • Fiabilité : 100%
  • Important : Il n'y a aucune raison de ne pas utiliser les micro-data. Les navigateurs ne les gérant pas vont simplement les ignorer.

Web Storage

Local Storage et Session Storage sont deux espaces de stockage associatif (clé-valeur) mis à disposition par le navigateur. Un espace unique est présent pour chaque domaine.

  • Local Storage : persistant
  • Session Storage : ne dure que le temps d'une session

On retrouve la fonctionnalité offerte par les cookies, en plus simple à utiliser, et supportant de plus grosses quantités de données, mais attention : les entrées ne sont pas accessibles par le serveur.

Capture

Lire et écrire des entrées

Pour lire :

var valeurLue = localStorage["cle"];

Pour ecrire :

localStorage["cle"] = valeur;

Base WebSQL

Une base de données relationnelle SQL est mise à disposition par le navigateur.

La base de données SQL est :

  • Locale
  • Associée à un domaine (pas d'accès cross-domain)
  • Persistante
  • Basée sur SQL Lite

Capture

Manipuler la base

Créer la base de données

database = openDatabase("nom", "wersion", "description", taille en o);

Executer des requêtes

this.db.transaction(
    function(tx) {
        tx.executeSql(query, ["values","to","replace","?","in","query"],
        function(tx, results){
            // SUCCESS
        },
        function(tx, error){
            // ERROR
        });
    }
)

Créer la base de données
